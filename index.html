Thank you for using PHP App Portal, below is full documentation of how to install and operate your site

Requirement

PHP App Portal has a few system requirements. All of these requirements are satisfied by our recommended hosting, so it's highly recommended that you use this hosting. However, if you had have your own, you will need to make sure your server meets the following requirements:

  • PHP >= 7.2.5
  • BCMath PHP Extension
  • Ctype PHP Extension
  • Fileinfo PHP extension
  • JSON PHP Extension
  • Mbstring PHP Extension
  • OpenSSL PHP Extension
  • PDO PHP Extension
  • Tokenizer PHP Extension
  • XML PHP Extension

Installation

All these steps has been recorded in the video below

  1. Extract .zip file downloaded from CodeCanyon. After extraction you will see 3 folders: "app", "update" and "documentation".
  2. Connect to your FTP account and upload all files in "app" folder to your root directory (please do not upload to public folder, view picture below).
  3. Rename "public_html" folder (you have uploaded) to the public folder name that determined by your hosting (e.g. httpdocs, public, www, htdocs).
  4. Create a database and assign a database user for it.
  5. Go to http://yourwebsite.com/install and fill out 3 forms, then you have done!

Category & Platform

PHP APP Portal separate application into two types, application and game, the categories also. Both have the same structure with:

  • Title The name of the category
  • HTML Title (Prefix & Suffix) The name of the category that will be show in some place on the website, you can use html here so it can be decorated with icon, font-style, etc.
  • Slug How the url looks like, example: House & Home will have the slug is house-home, this is good for SEO and user friendly.
  • App type The type of application that will be show in Google Search Result (Image below), this is not available for game because there is only one type of game in SERP
  • Order Display order, You can change this later by drag and drop in the listing

Platform contains both game and application so there is only 1 page for Platforms, just like categories, one platform also have 4 field:

  • Title Platform title (Ex: Android, Linux)
  • HTML Title (Prefix & Suffix) HTML display for the title (Ex: prefix: <strong> and suffix: </strong>
  • Slug Platform url will be displayed when browsing
  • Order The order belongs other platforms, this can be changed later by drag n drop in the listing

Application & Game

Application and game have a same structure, they were separated into 2 so that site user can easily browsing the site, just like Google Store or Apple Store. This is also easier for you to manage site content. From Dashboard menu, you can navigate to listing application or game through Application > All Applications or Games > All Games. The listing page is friendly and easy to understand, you can sort and filter apps by time created, is featured or not, by platform or by category.

To add new app (game), press "Add new App" Button on the top of the page (There is also 1 button at the bottom). One form will appear for you to fill:

  • Title: Title of the app
  • Slug: How the app displayed in the address. example: http://yourwebsite.com/app/android-things-toolkit
  • Icon: Icon URL, you can upload your own by pressing UPLOAD button next to it or insert URL from elsewhere.
  • Category & Platform: Select category and platform which the app is belong to
  • URL Link: The url can be direct link (for download button), webpage (Visit link) or buy page (Buy now button)
  • Button type: How button appear in single display app page, it could be DOWNLODAD, VISIT or BUY NOW
  • Author & Author link: App or Game has a author or developer, put those name & webpage here
  • Filesize: Size in Mb or Gb, if you do not fill in this field, Unknow file size will be displayed
  • Price: Price of app, if the app is free put 0 here
  • Screenshot: Screenshots of app, just like icon field, you can upload your own or put in URL, multi screenshots is available and separated by ";"
  • Short description & Description: Short description will be display in listing page and meta tag, full description will be displayed in single app page. Both support html

Pages

PHP App Portal allow you to generate custom page that support html. You can create about us page, policy page, term of services, etc. After you create your page, you can add them by editing footer which mentioned at customization section. A pages contain a title, an URL slug and a content block. The URL Slug will be generate automatically after you submit title, the content is html support with WYSIWYG editor, you can turn this off and enter your html code.

Google Play Grabber

PHP App Portal come with a built-in grabber, you can grabbe App or game from Google Play Store Automatically or Manually (Grabbe then edit with your own content).
To use this grabber, navigate to Google Play Grabber from side menu. You will see a list of apps and games, this is newest games from the store. This list is not affected by the location settings, the store will get apps from country that same IP with your hosting/server. To search, type keyword in search box then hit Enter.
In each app displayed, there is 3 buttons, Submit as App, Submit as Game and Quick Submit. First two buttons will redirect you to a form which all fields were filled, you just have to edit by your own then hit Submit. The third button will submit the app automatically with 100% content from Google Play, but you can change this later.
To get those content faster. There is also a button to quick grabbe all those apps at the top right corner of the listing page. This will trigger "Quick submit button" one by one.
